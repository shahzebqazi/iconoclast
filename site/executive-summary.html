Executive summary

Iconoclast Audio — intent and alignment

Status: Work in progress. Public narrative; no secrets.

What this is

Iconoclast Audio is a public-facing hub for audio engineering work: experiments, releases, and how the work is approached. Engineering-heavy source and CI may live elsewhere (for example GitLab); this GitHub presence is what ships to the open web—README, documentation, and this site.

Audience

  • People who care about sound quality, process, and tools.
  • Collaborators and future contributors who want context before diving into repos or mirrors.

Goals (near term)

  • A clear public landing on GitHub and a longer narrative in this summary.
  • A static site under site/ (HTML) so GitHub Pages serves real pages, not raw Markdown.
  • Later: a TypeScript-based GitHub Pages site (see repo docs/agent-prompt-typescript-github-pages.md).

Repo mechanics: Contributors & automation · README on GitHub

Product alignment (defaults for the next build)

  • Brand: Iconoclast Audio — audio engineering, not a generic SaaS landing.
  • Tone: Direct, technical, honest about WIP; no hype, no fake metrics.
  • Scope v1: Static content, accessible pages, no user accounts or API in the first TypeScript slice.
  • Hosting: GitHub Pages from this repo’s default branch; site output is the site/ directory (deployed via GitHub Actions).

Out of scope (for now)

  • Extra GitHub Actions beyond Pages deploy (add when there is a concrete workflow).
  • Cursor or “glass UI” telemetry in CI—only document if you introduce it deliberately.
