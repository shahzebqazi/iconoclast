Contributors & automation

How this repository is organized and how the public site is deployed.

Human vs. this page

Visitor-facing copy is in the README. This page summarizes what contributors and automation need to know.

Layout

  • site/ — Public website (HTML + CSS) deployed to GitHub Pages.
  • docs/ — Markdown source for long-form docs (edit on GitHub).
  • docs/executive-summary.md — Source for the executive summary page.
  • docs/agent-prompt-typescript-github-pages.md — Prompt for a future TS site.
  • README.md — GitHub landing page.
  • AGENTS.md — Full contributor notes in Markdown (repo root).
  • assets/readme-banner.svg — README banner image.

GitHub Pages

The site is published from the site/ directory using GitHub Actions (workflow .github/workflows/pages.yml) and the “GitHub Actions” source in Pages settings.

Do not rely on “Deploy from branch / (root)” for the live site — the canonical public pages are the HTML files under site/.

Conventions

  • Public-safe content only; no secrets in commits.
  • Keep the tone aligned with the executive summary (direct, technical, WIP-friendly).
